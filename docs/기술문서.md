# 프롬프트 빌더 — 기술 문서 (유지보수용)

비개발자용 사용법은 루트 [`README.md`](../README.md) 참고. 이 문서는 관리·확장용 상세.

## 1. 스택
- 백엔드: **Python + FastAPI + uvicorn**
- 프론트: **정적 HTML/CSS/JS** (빌드 도구 없음)
- 저장: `settings.json`(설정) · `archives.json`(보관함) · `presets/`(프리셋 폴더)
- 모델 호출: **OpenAI 호환 게이트웨이** (회사 LiteLLM 또는 샤오미 MiMo 직접), `openai` Python SDK로 `base_url`만 교체
- 실행: `setup.bat`(venv+의존성) / `run.bat`(서버+브라우저). 포트 **8770**

## 2. 폴더 구조
```
app.py                  FastAPI 진입점(정적 서빙 + 라우터 + 브라우저 오픈)
core/constants.py       경로·포트·기본 모델·프로바이더 프리셋
core/config.py          settings.json 로드/저장(.env 폴백, 키 마스킹)
services/llm.py         chat_reply / analyze_image / test_connection
services/prompt_spec.py 시스템 프롬프트 + 마감 지시 조립
services/manuscript.py  원고 파일 → 텍스트 추출(PDF/문서)
services/presets.py     프리셋 폴더 스캔 + 스타일 카탈로그 파싱
services/archive.py     보관함(archives.json) CRUD
routes/                 chat / settings / preset / manuscript / archive
prompts/                brand_guide.md · style_catalog.md(15종)
presets/                프리셋 폴더(아래 규약)
static/                 index.html · app.css · app.js
```

## 3. 환경변수 (`.env`, `.env.example` 참고)
| 키 | 설명 | 예시 |
|---|---|---|
| `UBION_LITELLM_URL` | 게이트웨이 URL | `http://localhost:4000` (사내 주소로 변경) |
| `UBION_LITELLM_KEY` | API 키 | `sk-...` |
| `CHAT_MODEL` | 텍스트 모델 | `deepseek-v4-flash` |
| `VISION_MODEL` | 비전 모델 | `MiMo-V2.5` |
| `HOST` / `PORT` | 서버 | `127.0.0.1` / `8770` |

설정 UI에서 바꾼 값은 `settings.json`에 저장되어 `.env`보다 **우선**.

## 4. 프로바이더 (설정 → 토글)
| 프로바이더 | base_url | chat | vision |
|---|---|---|---|
| 회사 LiteLLM | `http://localhost:4000` (사내 주소) | `deepseek-v4-flash` | `MiMo-V2.5` |
| 샤오미 MiMo(직접) | `https://api.xiaomimimo.com/v1` | `mimo-v2.5` | `mimo-v2.5`(멀티모달) |

- 신 OpenAI 모델(`gpt-5.5` 등)만 `max_completion_tokens` 사용, 그 외는 `max_tokens`(`core/constants.NEW_OPENAI_MODELS`).
- 「연결 확인」(`POST /api/test-connection`)은 두 모델에 초소형 ping.

## 5. 모델 라우팅 (`services/llm.py`, `routes/chat_routes.py`)
- 텍스트 턴 → `chat_model`.
- 이미지 첨부 턴(자유 빌더) → `vision_model`로 분석 텍스트 추출 후, `chat_model`이 대화/산출.
- **프롬프트 생성기는 원고를 텍스트로 추출**해 넣으므로 비전 불필요 → 텍스트 모델 단독.
- **빈 응답 처리**: 추론형 모델(MiMo 등)이 응답 토큰을 추론에 소진하면 `content`가 빌 수 있음 → `reasoning_content` 폴백, 그래도 비면 `ok:false`로 사유 반환. 토큰 상한은 `chat_reply`의 3500.

## 6. 원고 텍스트 추출 (`services/manuscript.py`, `POST /api/extract-text`)
- 지원: **PDF**(`pypdf`), DOCX·HWPX·PPTX(표준 라이브러리 zip+xml), TXT/MD/CSV/JSON.
- 한계: **스캔/이미지 PDF는 추출 불가**(OCR 미지원), PPTX는 도형·순서가 흩어질 수 있음(best-effort) → 부실하면 메모란에 직접 붙여넣기.
- 상한 2만자(`MAX_CHARS`).

## 7. 프리셋 (`services/presets.py`)
두 종류가 합쳐져 목록(`GET /api/presets`)으로 노출:
1. **폴더 프리셋** — `presets/<이름>/` (카드뉴스 묶음, 본인 완료본)
2. **스타일 15종** — `prompts/style_catalog.md` 표를 파싱(단일 출처, `kind:"style"`)

폴더 규약:
```
presets/<이름>/
  meta.json   {"description","keywords":[],"builtin":true,
               "purpose","tone","color","style","cuts"}  # 옵션 매핑
  preset.md   설명/프롬프트 골격(선택)
  cover.png   썸네일(선택)
```
- `builtin:false`(UI 「현재 옵션을 프리셋으로 저장」)만 삭제 가능.
- 프리셋의 `purpose/tone/color/style/cuts`가 생성기 버튼을 자동 선택.

## 8. 보관함 (`services/archive.py`, `/api/archives`)
- `archives.json`에 `{id, title, created, content}` 저장(최신순). 목록은 미리보기·글자수만, 상세는 `GET /api/archives/{id}`.
- 프런트 「📌 보관」 → `POST`, 게시판에서 열기/복사/.md/삭제.

## 9. API 요약
| 메서드 | 경로 | 설명 |
|---|---|---|
| POST | `/api/chat` | 멀티모달 1턴(+`finalize`) |
| GET/POST | `/api/settings` · POST `/api/test-connection` | 설정·연결확인 |
| GET/POST/DELETE | `/api/presets` `/api/presets/{name}` | 프리셋 |
| POST | `/api/extract-text` | 원고 추출 |
| GET/POST/GET/DELETE | `/api/archives` `/api/archives/{id}` | 보관함 |

## 10. 트러블슈팅
| 증상 | 원인/해결 |
|---|---|
| 연결 타임아웃 | 회사 LiteLLM은 사내망 전용. MiMo 직접은 외부망. |
| 결과 빈 화면 | 모델 빈 응답 → 재시도, 또는 텍스트 모델을 `deepseek-v4-flash`로. |
| 모델 not found | 설정의 모델명(등록 별칭) 확인. |
| 코드 바꿨는데 반영 안 됨 | `.py`→`run.bat` 재시작, 화면→`Ctrl+Shift+R`. 의존성 변경 시 `setup.bat`. |
